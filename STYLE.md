# design.md - “神秘数字”转换器前端设计规范

## 1. 行业通用设计原则 (Industry Design Principles)

* **渐进增强与优雅降级 (Progressive Enhancement)**：核心功能在任何设备上都必须可用。高端设备展示完整的 WebGL 流体特效，低端设备或省电模式下无缝降级为 CSS 静态渐变。
* **无障碍与可读性 (A11y & Readability)**：确保前景文字与动态背景之间存在足够的对比度。输入框和结果展示区采用模糊背景（毛玻璃），将用户的视觉焦点牢牢固定在信息层。
* **格式塔原理 - 亲密性 (Gestalt - Proximity)**：将强相关的元素（平台切换 Toggle、输入框、转换按钮）在视觉上紧密编排在一个“云朵”卡片内，降低用户的认知负荷。
* **明确的系统状态反馈 (Visibility of System Status)**：任何交互（如检索中、转换成功、无匹配结果）都必须伴随柔和的视觉反馈（如按钮的果冻弹跳或看板娘的表情切换）。

## 2. 核心视觉语言 (Core Visual Language)

 * **视觉关键字**：可爱 (Cute)、云朵 (Cloud)、棉花 (Cotton)、呼吸感 (Airy)。
 * **形状与边缘**：拒绝生硬的直角和圆形。所有交互元素均采用不规则的“云朵状”平滑边缘（使用复杂组合的 `border-radius`，例如 `50% 50% 50% 50% / 60% 60% 40% 40%`）。
 * **色彩基调**：主色调采用浅马卡龙色系。背景色在加载漫画封面后，应**提取封面主色调并作为背景渐变色**，配合流体动画实现自然过渡。
 * **排版与图标**：优先使用圆体（如 Varela Round）。**所有图标必须使用 FontAwesome，严禁使用 Emoji**。字体字号偏大，行距宽松。

## 3. WebGL 动态背景专项方案 (WebGL Architecture)

为了满足“兼容性好、性能占用低、可读性高”的三大硬性指标，背景的 WebGL 渲染采取以下技术策略：

* **视觉效果 (保障可读性)**：不使用带有清晰轮廓的 3D 模型或高频闪烁的粒子。采用基于 Fragment Shader (片段着色器) 编写的 **2D 柔和流体渐变 (Fluid Gradient)** 或缓动的云雾效果。
* **性能优化 (保障低占用)**：
    * **限制帧率**：背景动画不需要 60fps，将其限制在 24fps - 30fps 即可保持柔和的动感，大幅降低 GPU 功耗。
    * **按需渲染**：监听浏览器的 `visibilitychange` 事件。当用户切换到其他标签页或缩小窗口时，立即调用 `cancelAnimationFrame` 暂停渲染。
    * **降低分辨率**：将 WebGL Canvas 的内部渲染分辨率设定为设备物理分辨率的 0.5 到 0.75 倍，通过 CSS 放大铺满屏幕，这在渲染柔和渐变时不会产生明显的马赛克，但能节省极大的算力。
* **兼容性保证 (保障极佳兼容)**：强制使用 WebGL 1.0 基础上下文，避免使用 WebGL 2.0 的高级特性以确保兼容老旧的移动端浏览器。

> **背景降级策略矩阵表**

| 运行环境状态 | 背景渲染方案 | 预期表现 |
| :--- | :--- | :--- |
| 高性能设备 + 正常模式 | WebGL Fragment Shader | 柔和的流体云朵渐变循环，带有极轻微的鼠标光晕跟随。 |
| 低端设备 / 移动端低电量 | CSS3 `@keyframes` 动画 | 基于 DOM 的超大尺寸径向渐变（Radial Gradient）缓慢平移。 |
| 系统开启 `prefers-reduced-motion` (减弱动态效果) | 静态 CSS 背景 | 渲染单张生成的马卡龙渐变图，完全关闭所有浮动和背景动画。 |

## 4. 响应式布局架构 (Responsive Layout)

网站采用绝对的响应式设计（Mobile-first 理念 + Desktop 特化）。

* **横屏 / 桌面端 (Landscape / Desktop)**：
    * 采用不对称的左右交叠布局。
    * **左侧视觉区 (占宽 ~40%)**：默认展示网站占位看板娘立绘。检索后展示漫画封面。图片滑入带有淡入淡出（Fade-in + Slide-up）效果。右侧边缘应用 CSS Mask（蒙版遮罩）实现透明度渐变，与右侧自然交叠。
    * **中央操作区 (占宽 ~30%)**：检索面板，采用云朵状边缘的卡片设计。
    * **右侧换乘区 (占宽 ~30%)**：检索成功后以 slideInRight 动画从右侧滑入出现，与中央操作区并列。未触发时该区域不渲染。
* **竖屏 / 移动端 (Portrait / Mobile)**：
    * 采用上下堆叠布局，优先保证单手操作的便利性。
    * **上半区 (占高 45-50%)**：展示立绘或封面，底部边缘使用线性渐变蒙版（Linear-gradient mask）向下渐渐融入背景。
    * **中部**：输入与操作区块放置在屏幕中下方，方便拇指触达，边缘保持云朵化处理。
    * **下部**：换乘面板在检索成功后以 slideUpFade 动画向下堆叠出现。

## 5. 组件与交互细节 (Components & Interactions)

 * **全局导航 (Header)**：左上角悬浮汉堡菜单（使用 FontAwesome 图标），紧挨圆润字体 Logo。点击后弹出独立悬浮菜单面板。
 * **操作面板内容**：包含数据源选择 Toggle（API代理 / 前端直连）、云朵形状无边框输入框、带有果冻弹跳动效的转换按钮。
 * **换乘面板 (Transfer Panel)**：
   * 与操作面板使用完全相同的 `glass-card` 毛玻璃云朵卡片组件。
   * 顶部内嵌 Tab 切换栏（使用与平台选择 Toggle 相同的 `platform-btn` 样式），根据输入源动态显示可用的换乘目标平台。
   * 内容区显示模糊搜索结果列表，每条结果包含缩略图、标题、平台来源标识，可点击跳转。
   * 加载中、空状态、错误状态均复用全局的 `fade-element` 过渡系统。
 * **全局动效与物理互动 (Physics & Micro-interactions)**：
   * 所有主要悬浮区块除了带有半透明毛玻璃属性外，在鼠标悬浮（Hover）时，应监听鼠标的相对位置，产生**3D 物理倾斜**（CSS `transform: perspective(...) rotateX(...) rotateY(...)`）。
   * 默认附带微弱的平滑位移动画，模拟失重感。
