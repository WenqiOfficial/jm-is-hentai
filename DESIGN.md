 # Hentai车站 网站设计规范

 ## 视觉设计

 ### 颜色方案
 - 主色调：浅马卡龙色系（婴儿蓝、樱花粉、奶油黄等）
 - 背景色：默认缺省浅色渐变，根据漫画封面主色调动态取色并平滑过渡
 - 文字色：黑色 (#1e293b) 或深灰色 (#64748b)
 - 强调色：根据背景动态调整，确保足够的对比度

 ### 字体与图标
 - 主字体：系统默认字体栈中的圆体优先 (Varela Round, Quicksand, sans-serif)
 - 图标：统一使用 FontAwesome 图标库，**严禁使用 Emoji**，保持视觉一致性
 - 标题：使用加粗圆体，字号根据层级递减 (H1: 32px, H2: 24px, H3: 20px)
 - 正文：16px，行高 1.6

 ## 用户界面布局

### 页面结构
 - 顶部导航栏：包含网站标题、主要功能入口
 - 主内容区：输入框、结果显示区域
 - 底部信息栏：版权信息、链接等

### 输入与操作区域
 - 单行输入框：用于输入 JMComic 编号，大圆角无边框设计
 - 搜索按钮：如果冻般 Q 弹的按钮交互
 - 设置菜单：提供数据源切换选项（API 代理 / 前端内置直连）

### 结果展示
 - 云朵卡片布局：搜索结果以不规则的云朵状卡片展示
 - 包含封面图、标题、作者、标签等关键信息
 - 明确的跳转按钮指向不同平台
 - 加载状态：具有呼吸感或旋转动画的加载指示器

### 换乘面板 (Transfer Panel)
 - **触发条件**：仅在检索成功获得结果后才出现
 - **桌面端位置**：在检索面板的右侧，作为独立的云朵卡片浮现（slideIn 动画从右侧滑入）
 - **移动端位置**：在检索面板的下方堆叠展示
 - **视觉风格**：与检索面板保持完全一致的云朵卡片毛玻璃设计，相同的 3D 倾斜微交互
 - **内部结构**：
   - 顶部 Tab 导航：根据当前检索平台动态显示可用的换乘目标平台
   - 当输入源为 **JMComic** 时：显示 `E-Hentai` 和 `哔咔漫画` 两个 Tab
   - 当输入源为 **E-Hentai** 时：显示 `JMComic` 和 `哔咔漫画` 两个 Tab
   - 每个 Tab 下展示模糊检索的结果列表（标题、缩略图、相关度），可直接点击跳转
   - 无结果时显示友好的空状态提示

## 交互设计

### 响应时间
 - 页面加载：首屏瞬间加载
 - 搜索响应：支持多备用域名轮询，提高请求成功率
 - 错误反馈：友好的错误提示

### 微交互与物理反馈 (Micro-interactions)
 - 鼠标悬浮：当鼠标经过卡片或元素时，产生基于鼠标位置的 3D 物理倾斜和细微的位移反馈
 - 成功状态：柔和的界面元素展开
 - 加载状态：毛玻璃效果下的元素渐变

## 可访问性

 - 支持键盘导航
 - 足够的对比度（至少 4.5:1）
 - 语义化的 HTML 标签
 - ARIA 属性增强辅助功能

 ## 响应式设计

 - 移动端优先设计
 - 断点设置：
   - 手机：< 640px
   - 平板：640px - 1024px
   - 桌面：> 1024px
 - 自适应布局，确保在各种设备上都能正常浏览

 ## 性能优化

 - 静态资源压缩
 - 图片懒加载
 - API 请求缓存
 - CDN 加速

 ## 错误处理

 - 网络错误：友好的错误提示，建议重试
 - 数据错误：明确指出问题所在
 - 输入错误：实时验证并给出修改建议

 ## 数据展示规范

 ### 漫画信息
 - 标题：完整显示，过长时省略
 - 封面：统一尺寸 200x280px，保持比例
 - 作者：清晰标识
 - 标签：以标签云形式展示

 ### 搜索结果
 - 按相关性排序
 - 显示平台来源标识
 - 包含评分或热度信息（如果可用）

 ## 功能模块设计

 ### 搜索模块
 - 支持 JMComic ID 查询
 - 双数据源实现：
   - 后端 API 代理模式（Cloudflare Workers，默认推荐）
   - 前端内置直连模式（省流/直接在客户端发送请求）
 - 支持多域名回退（Fallback）机制

 ### 多语言与标签翻译 (i18n & Tag Translation)
 - 全局 UI 支持中/英/日三语无缝切换
 - 基于 `EhTagTranslation` 数据库，实现漫画标签的本地化翻译
 - 本地化数据库由 IndexedDB 缓存，支持手动静默更新

 ### NSFW 保护
 - 提供“自动模糊 NSFW 封面”的设置开关
 - 当检索结果的标签包含 R-18/NSFW 相关标记时，封面将自动覆盖强模糊层与提示文字
 - 用户主动点击后方可显示原图

 ### 换乘模块 (Cross-Platform Transfer)
 - **触发**：检索成功后自动执行跨平台模糊搜索
 - **搜索策略**：使用检索到的漫画标题、作者名作为关键词，在目标平台进行模糊匹配
 - **支持平台**：
   - E-Hentai：通过标题关键词搜索 e-hentai.org
   - 哔咔漫画 (PicACG)：通过标题关键词搜索
   - nHentai：通过标题直接拼接 nhentai.net/search/ URL (无需调用外部API)
   - JMComic：当从 E-Hentai 入口进入时，反向搜索 JMComic
 - **结果展示**：每个平台的匹配结果以列表形式展示在独立的 Tab 页内
 - **跳转**：每条结果均提供直接跳转链接

 ### 结果对比模块（未来扩展）
 - 并排展示不同平台的结果
 - 高亮差异点
 - 提供收藏功能

 ---

 以上设计规范将指导整个项目的视觉和交互设计，确保用户体验的一致性和友好性。
